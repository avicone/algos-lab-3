# algos-lab-3

# Система обработки задач (Queue)

Реализация приоритетной очереди задач на Python с использованием `collections.deque`. Задачи автоматически сортируются по приоритету при добавлении — чем меньше число, тем выше приоритет.

## Особенности

- **Приоритетная вставка** — задачи с более высоким приоритетом (меньшее число) автоматически размещаются ближе к началу очереди
- **Типобезопасность** — проверка типа при добавлении (только объекты класса `Task`)
- **Стандартные операции очереди** — FIFO с учётом приоритета
- **Информативный вывод** — наглядное отображение содержимого очереди

## Структура проекта

### Класс `Task`
Представляет отдельную задачу.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `task_id` | `int` | Уникальный идентификатор задачи |
| `description` | `str` | Описание задачи |
| `priority` | `int` | Приоритет (0 — наивысший) |

### Класс `TaskQueue`
Управляет коллекцией задач на основе `deque`.

| Метод | Описание |
|-------|----------|
| `enqueue(task)` | Добавляет задачу с учётом приоритета |
| `dequeue()` | Извлекает задачу с наивысшим приоритетом |
| `front()` | Просмотр задачи без извлечения |
| `isEmpty()` | Проверка на пустоту |
| `size()` | Количество задач в очереди |
| `clear()` | Полная очистка очереди |
| `display()` | Отображение всех задач |

## Быстрый старт

### Использование

```python
from task_queue import Task, TaskQueue

# Создание очереди
queue = TaskQueue()

# Добавление задач
queue.enqueue(Task(1, "Исправить критический баг", 0))
queue.enqueue(Task(2, "Обновить документацию", 2))
queue.enqueue(Task(3, "Добавить фичу", 3))

# Просмотр очереди
queue.display()

# Получение задачи с наивысшим приоритетом
next_task = queue.dequeue()
print(f"Выполняется: {next_task}")
```

### Пример вывода

```
==============================================================
Очередь задач (всего задач: 3)
==============================================================
1. Задача #1 | Приоритет: 0 | Описание: Исправить критический баг
2. Задача #2 | Приоритет: 2 | Описание: Обновить документацию
3. Задача #3 | Приоритет: 3 | Описание: Добавить фичу
==============================================================
```

## Логика приоритетов

Приоритет определяется по возрастанию: **0 — наивысший приоритет**, затем 1, 2, 3 и так далее.

```
Приоритет 0 (критический) → выполняется первым
Приоритет 1 (высокий)
Приоритет 2 (средний)
Приоритет 3 (низкий)
Приоритет 4 (фоновый) → выполняется последним
```

## Сложность операций

| Операция | Временная сложность |
|----------|---------------------|
| `enqueue` | O(n) — поиск позиции вставки |
| `dequeue` | O(1) — извлечение из начала |
| `front` | O(1) — доступ к первому элементу |
| `isEmpty` | O(1) |
| `size` | O(1) |

## Обработка ошибок

- При попытке добавить объект не типа `Task` выбрасывается `TypeError`
- При извлечении из пустой очереди выводится предупреждение и возвращается `None`
- При просмотре пустой очереди через `front()` выводится сообщение
